DataWiper

Le contexte

La MOA à régulièrement besoin d'intéroger les BDDs pour répondre aux demandes du métier.
Afin d'être plus autonome sur ce sujet un outils était nécessaire.
Le projet restera interne, donc une application restreinte.

Les Spécifications fonctionnelles

Interface WEB pour faciliter l'accès et les échanges.
Sauvegarder, consulter, modifier, paramètrer, executer des requetes SQL.
Une maquette as été établie dans un but d'organisation.

Les Spécifications techniques

Une architecture trois tiers afin de rendre le développement + rapide, + maintenable et + fiable.
Avec le modèle de conception MVC (Model View Controller)
Une base de données pour la persistance avec SQLite.
Technologies et composants utilisées: TypeScript, Ag-Grid, CodeMirror, JQuery, ASP.Net Core, ADO.Net

Conclusion

Aujourd'hui le SQLTheque, c'est environ 200 script SQL de stocker et de partager sur l'intranet.
Une interface simple pour consuler et trier les scripts
Un éditeur SQL pour ajouter où modifier les scripts
Une zone pour gérer les paramètres optionnels.
Et enfin un écran pour executer avec des paramètres renseigner au choix.
